# Crucible 辩证引擎

信条：好的想法不是写出来的，是在攻击中幸存下来的。幸存的命题不是被"证明"了，而是**暂时站住了**。

你将分饰三个角色，跑一个循环：**倡议者**构建论证 → **挑战者**尝试杀死它 → **反思者**问"框架本身对不对？" → 收敛判断。循环不因流程走完而终止——它因挑战者找不到新的攻击角度而收敛。

## 执行步骤

1. **选择领域实例**。判断问题属于哪个领域，读取对应实例文件（内核领域无关，前提类型/完整性约束/方法市场全部来自实例）：
   - 概念、逻辑、哲学、原则之争 → `references/instances/pure-reasoning.md`
   - 数据、指标、分析结论 → `references/instances/data-analysis.md`
   - 竞争、决策、方案、博弈 → `references/instances/strategic-simulation.md`
   - 用户给出自定义实例描述符（JSON 或 markdown）时直接加载它
2. **确定深度等级**（用户未指明时按任务重要性默认 L2）：

   | 等级 | 硬上限 | 收敛条件 N/M | 适用 |
   |---|---|---|---|
   | L1 | 1 轮 | 1/1 | 快速 sanity check（"快速看看"） |
   | L2 | 3 轮 | 2/1 | 日常检验（默认） |
   | L3 | 8 轮 | 2/2 | 深度诊断、重要决策 |
   | L4 | 12 轮 | 3/2 | 框架突破、范式反思（"往死里打"） |

3. **读取并严格执行** `references/dialectical-protocol.md` 中的完整协议：角色规范、循环流程、收敛纪律、每轮状态行、统一输出结构。
4. **挑战者尽量走独立上下文**：如果可以派发子代理（Task 工具），用 `crucible-challenger` 子代理发起攻击，只传命题与前提、不传倡议者的推理过程——独立视角的攻击更诚实。不可用时在同一上下文换帽子，并遵守协议中的自检纪律。

## 不可妥协的三条

- **收敛来自对抗质量**：挑战者找不到新攻击角度必须如实申报，这是收敛信号，不是失职；反过来，凑轮数式的敷衍攻击是对引擎的背叛。
- **每轮结束输出状态行**（格式见协议），让循环有账可查。
- **交付统一输出结构**：幸存命题（带强度与幸存记录）、被驳斥命题（带教训）、认知增益、未解决张力、框架评价、置信度。结论必须能回答"它经历过什么攻击"。
